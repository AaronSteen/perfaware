    now we are back in WSL. what has happened so far: cmd.exe /c "$SCRIPT_WIN" 2>&1. that's all.

    cmd.exe /c "$SCRIPT_WIN" 2>&1 | <---- this pipe is what we're interpreting next. the output of the 
        batch script we analyzed above is piped into sed. we're expecting error messages
        from the compiler that will reference specific file paths for code files in the project.
        the file paths will be in windows format since we switched to command prompt to compile.

        -e 's|\r||g' \
            - "-e" specifies a script or command that will be executed. since we are
                running multiple commands, we need it
            - s means substitute
            - | is delimiter
            - \r is the character for which we want to substitute something. /r is the carriage
                return character that windows uses but linux doesn't.
            - || is the character we want to substitute for /r (we want to substitute 
                nothing; we want to delete /r)
            - g means do it for every occurrence on a line, not just the first occurrence
            - \ means continue the command on the next line

        -e 's|C:\\Users\\ams56\\work\\|/mnt/c/Users/ams56/work/|g'
            - "-e" means here's another command
            - s is substitute
            - | is delimiter
            - C:\\Users\\ams56\\work\\ is the windows path, replace it
                with /mnt/c/Users/ams56/work/ so neovim quickfix list can know where the files with 
                errors are. neovim wouldn't be able to interpret the windows file paths since we're
                running inside wsl. /mnt/c/Users... is how programs running in WSL understand
                files in the windows filesystem.

        - e 's|\\|/|g'
            replace all instances of \\ with / that bash wants

        -e 's|(\([0-9][0-9]*\),\([0-9][0-9]*\))|:\1:\2|g'       
            we want to match error lines emitted by clang that are like this:

                C:\Users\ams56\work\beaver\code\beaver.cpp(142,17): error: use of undeclared identifier

            and convert the line and character part portion (142,17) to :142:17 so that neovim can
                understand it. therefore some regex to match (142,17) and replace the ( and )
                characters with : characters.
        
            ( <-- literal parenthesis
                \( <-- start a regex capture group (first capture group, can later reference it with "/1"
                    [0-9][0-9]* <-- match one digit, followed by zero or any number of digits.
                        e.g., it would match:
                            1, because match one digit and 0 digits follow this 1 
                            14, because match one digit (the 1), and the 4 because it comes after the 1 
                            142, because match the 1 and any number of subsequent digits.
                            etc. it would also match 1465498464059.

                \) <-- end the capture group
                ,  <-- match a literal comma
                \( <-- start a new capture group
                    [0-9][0-9]* <-- as above. match one digit followed by zero or any number of digits.
                \) <-- end the capture group
            ) <-- literal parenthesis.

            thus we have matched (142,17), (3694,8), (1, 23) etc.

            after the | delimiter, we have another expression specifiying the string we want to replace the characters
                we just matched. basically we delete what was matched and put in its place what comes after the delimiter.

                :\1:\2

                    : <-- put a : character
                    \1 <-- this means put capture group 1, so the first set of one or more digits we matched. e.g., 142
                    : <-- put another : character
                    \2 <-- this means put capture group 2. second set of numbers. e.g., 17

exit ${PIPESTATUS[0]}

    exit the bash script, but instead of exiting with whatever error code sed produced (almost always a 0 for success),
    index into a list of error codes stored in the PIPESTATUS.
    
    braces are mandatory because they tell bash to interpret the command PIPESTATUS[0] as a single command.
    without the braces, bash would read PIPESTATUS and interpret it, without referencing the [0] to index
    the first item in the PIPESTATUS array.
